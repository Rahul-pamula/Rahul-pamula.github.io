## 🚀 Professional Experience & Projects

### 1. 📧 Enterprise Email Platform – Custom Email Engine
**System Design & Backend Engineer** *(Jan 2026 – Present)*

#### 🔴 Problem Statement
Managing emails from multiple providers (SendGrid, AWS SES, Azure) with different APIs, rate limits, and requirements is complex. Organizations struggle with:
- Handling failed deliveries and implementing robust retry mechanisms
- Tracking campaign performance across multiple channels
- Managing duplicate sends and ensuring idempotency
- Scaling to handle millions of emails efficiently

#### 📋 Scenario
A growing SaaS company needs to send bulk emails to customers through multiple channels. They face challenges like:
- Marketing teams need to create targeted campaigns and segment users dynamically
- Users must track real-time delivery status, bounces, and user engagement
- The system must handle provider-specific constraints (rate limiting, quota management)
- Need reliable webhook tracking for opens, clicks, and deliveries

#### ✅ Solution Implemented
- **Scalable Multi-Tenant Architecture:** Architected using FastAPI, PostgreSQL, and Redis for high availability
- **Async Queue Processing:** Engineered asynchronous queue-based processing with RabbitMQ for reliable message delivery
- **Rate Limiting & Idempotency:** Implemented sophisticated rate-limiting to respect provider constraints and idempotent operations to prevent duplicate sends
- **Multi-Provider Integration:** Seamlessly integrated SendGrid, AWS SES, and Azure Mail services with provider failover
- **REST API Design:** Built comprehensive REST APIs for campaign creation, user segmentation, batch operations, and webhook tracking
- **Monitoring & Analytics:** Real-time dashboard for campaign performance, delivery rates, and user engagement metrics

**Tech Stack:** FastAPI | PostgreSQL | Redis | RabbitMQ | Docker | AWS | Azure

- No intelligent summarization or context understanding of conversations
- Real-time processing of unstructured data is challenging

#### 📋 Scenario
Imagine a project team using WhatsApp for collaboration. They receive:
- Text messages with project deadlines and technical requirements
- Images of design mockups and whiteboard sketches
- PDF documents with specifications and contracts
- Voice messages with meeting notes and decisions
- Mixed conversations in different languages

The team needs to automatically:
- Extract all deadlines and create reminders
- Understand image content (designs, diagrams, handwritten notes)
- Parse PDFs for key information
- Transcribe voice messages and extract action items
- Summarize key discussions and decisions

#### ✅ Solution Implemented
- **NLP-Powered Deadline Extraction:** Built intelligent deadline extraction using Google Gemini AI with context understanding
- **Multi-Modal AI Processing:** Integrated Azure AI Services for comprehensive analysis:
  - **Vision API:** Analyze images, recognize text, identify objects, and understand visual content
  - **Document Intelligence:** Extract structured data from PDFs, forms, and receipts
  - **Speech-to-Text:** Convert voice messages to text with high accuracy
  - **Text Analytics:** Sentiment analysis, key phrase extraction, and entity recognition
- **FastAPI Backend:** Production-ready backend with efficient pipeline for real-time processing
- **Intelligent Routing:** Smart message routing based on content type and priority
- **Context Awareness:** Maintains conversation context for better deadline and task extraction

**Tech Stack:** FastAPI | Google Gemini AI | Azure AI Services | Python | PostgreSQL | NLP Libraries
